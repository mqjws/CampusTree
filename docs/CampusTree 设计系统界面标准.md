# CampusTree Design System

## 1. 设计原则

CampusTree 的设计系统服务于一个以匿名表达、轻互动、内容优先为核心的校园社区。整体视觉应满足以下原则：

- 内容优先：弱化装饰性，强调标题、正文、互动信息的阅读效率。
- 匿名友好：减少强人格化设计，保持克制、平等、低压的社区氛围。
- 轻盈清新：延续校园与成长感，主色采用清新的绿色体系。
- 移动优先：优先保障移动端单手操作、触达效率和信息层级。
- 一致可扩展：颜色、间距、圆角、阴影、动效均通过统一设计令牌管理，便于后续主题扩展。

## 2. 颜色系统

### Primary
- HEX：`#10B981`
- RGB：`rgb(16, 185, 129)`
- 使用场景：品牌主色、主按钮、激活态、选中态、关键操作入口、链接高亮

### Secondary
- HEX：`#3B82F6`
- RGB：`rgb(59, 130, 246)`
- 使用场景：辅助操作、信息型标签、次重点入口、说明性图标

### Success
- HEX：`#22C55E`
- RGB：`rgb(34, 197, 94)`
- 使用场景：成功反馈、表单校验成功、发布成功、状态确认

### Warning
- HEX：`#F59E0B`
- RGB：`rgb(245, 158, 11)`
- 使用场景：风险提示、待确认操作、内容提醒、系统注意事项

### Danger
- HEX：`#EF4444`
- RGB：`rgb(239, 68, 68)`
- 使用场景：删除、退出登录、报错反馈、不可逆操作、点赞激活强调

### Info
- HEX：`#6366F1`
- RGB：`rgb(99, 102, 241)`
- 使用场景：中性提示、系统消息、说明型状态、加载过程中的信息反馈

### 中性色补充

虽然业务主色只有六类，但 CampusTree 必须配套中性色以保证可读性与卡片层次：

- 主文本：`#1F2937` / `rgb(31, 41, 55)`
- 次文本：`#6B7280` / `rgb(107, 114, 128)`
- 禁用文本：`#9CA3AF` / `rgb(156, 163, 175)`
- 页面背景：`#F9FAFB` / `rgb(249, 250, 251)`
- 卡片背景：`#FFFFFF` / `rgb(255, 255, 255)`
- 悬浮背景：`#F3F4F6` / `rgb(243, 244, 246)`
- 边框颜色：`#E5E7EB` / `rgb(229, 231, 235)`

### 颜色使用建议

- `Primary` 占视觉焦点，不超过单屏高饱和色面积的 20%。
- `Danger` 仅用于反馈和高风险操作，不作为常规品牌强调。
- 页面层级主要依赖中性色和阴影，不依赖大面积纯色分区。
- 所有正文与背景需满足至少 4.5:1 对比度。

## 3. 字体系统

建议字体策略：

- 中文优先：`PingFang SC`、`Noto Sans SC`、`Microsoft YaHei`
- 英文与数字：`Inter` 作为补充
- 回退：`sans-serif`

### H1
- 字体大小：`32px`
- 字重：`700`
- 行高：`1.25`
- 使用场景：页面主标题、品牌级模块标题、首页核心区标题

### H2
- 字体大小：`24px`
- 字重：`700`
- 行高：`1.33`
- 使用场景：二级页面标题、卡片区块标题、用户中心分组标题

### H3
- 字体大小：`18px`
- 字重：`600`
- 行高：`1.5`
- 使用场景：帖子标题、弹窗标题、设置项分组标题

### Body
- 字体大小：`16px`
- 字重：`400`
- 行高：`1.75`
- 使用场景：帖子正文、评论正文、表单内容、说明文本

### Caption
- 字体大小：`13px`
- 字重：`400`
- 行高：`1.5`
- 使用场景：时间、统计数据、辅助说明、标签补充信息

### 排版规则

- 帖子正文优先使用 `Body`，确保长文本舒适阅读。
- 移动端帖子标题可降至 `16px`，但正文不低于 `14px`。
- 数字信息如点赞数、评论数、时间统一使用 `Caption`。
- 同一模块中标题层级不超过 3 层，避免信息噪音。

---

## 4. 间距系统

### 4px
- 使用场景：图标与文字微间距、标签内部间距、极小型控件对齐修正

### 8px
- 使用场景：按钮内元素间距、卡片内短信息组、表单项内部补白

### 12px
- 使用场景：列表项内部间距、评论块内部间距、紧凑型模块分隔

### 16px
- 使用场景：卡片默认内边距、移动端页面左右边距、表单控件默认高度节奏

### 24px
- 使用场景：模块与模块之间、页面区块分隔、桌面端卡片内大分区

### 32px
- 使用场景：页面主区块上下留白、弹窗内容区间距、重要内容分层

### 48px
- 使用场景：页面首屏留白、大标题与内容区分隔、空状态和大模块呼吸感

### 间距使用规则

- 所有页面与组件只允许使用以上固定间距令牌。
- 移动端优先使用 `8px / 12px / 16px / 24px`。
- 桌面端页面结构优先使用 `24px / 32px / 48px`。

---

## 5. 圆角系统

### 4px
- 使用场景：输入框、小型按钮、小标签、轻量级状态块

### 8px
- 使用场景：默认按钮、评论卡片、下拉菜单、小型弹层

### 12px
- 使用场景：帖子卡片、用户信息卡片、表单容器、常规浮层

### 16px
- 使用场景：主卡片、大型弹窗、移动端底部抽屉、重点内容容器

### 圆角使用规则

- CampusTree 以中等圆角为主，不使用过于锐利或过于拟物的风格。
- 卡片体系默认优先 `12px`，按钮体系默认优先 `8px`。
- 同屏圆角层级不宜超过 3 种。

---

## 6. 阴影系统

### Card Shadow
- 阴影值：`0 2px 8px rgba(15, 23, 42, 0.08)`
- 使用场景：默认帖子卡片、用户信息卡片、基础信息容器

### Hover Shadow
- 阴影值：`0 8px 20px rgba(15, 23, 42, 0.12)`
- 使用场景：桌面端卡片悬浮、可点击模块 hover 状态、重点浮层预反馈

### Modal Shadow
- 阴影值：`0 20px 40px rgba(15, 23, 42, 0.18)`
- 使用场景：确认弹窗、发布弹层、设置弹层、风险操作对话框

### 阴影使用规则

- 移动端减少强阴影，优先使用边框和留白构建层级。
- Hover Shadow 仅在桌面端使用。
- 不叠加多层重阴影，避免界面显得脏和厚重。

---

## 7. 按钮规范

按钮统一分为四类：Primary Button、Secondary Button、Danger Button、Disabled Button。

### 通用尺寸

- Large：`48px` 高，适用于页面主操作、提交、发布
- Medium：`40px` 高，适用于常规操作
- Small：`32px` 高，适用于卡片内二级操作
- 左右内边距建议：`16px / 20px / 24px` 随尺寸递增
- 字体建议：`14px` 或 `16px`，字重 `500`

### Primary Button
- 背景色：`#10B981`
- 文字色：`#FFFFFF`
- 边框：无或同色
- 使用场景：发布、确认、提交、继续、主要 CTA

状态规范：
- Default：品牌主色实底
- Hover：`#059669`
- Active：比 Hover 再深一级，强调按压感
- Focus：带主色外发光或清晰描边
- Loading：保持主色，内容切换为加载反馈

### Secondary Button
- 背景色：`#FFFFFF`
- 文字色：`#374151`
- 边框色：`#D1D5DB`
- 使用场景：取消、返回、次级筛选、辅助操作

状态规范：
- Default：白底描边
- Hover：背景切为 `#F3F4F6`
- Active：边框与文字略微加深
- Focus：蓝绿系描边高亮

### Danger Button
- 背景色：`#EF4444`
- 文字色：`#FFFFFF`
- 边框：无或同色
- 使用场景：删除、退出登录、清空内容、危险确认

状态规范：
- Default：红色实底
- Hover：`#DC2626`
- Active：进一步加深，明确风险
- Focus：红色外描边提示

### Disabled Button
- 背景色：`#E5E7EB`
- 文字色：`#9CA3AF`
- 边框色：`#E5E7EB`
- 使用场景：不可提交、条件未满足、加载锁定时

状态规范：
- 不响应 Hover
- 不显示强交互反馈
- 禁止与 Primary 同时出现同级竞争关系

### 按钮使用规则

- 一个视图内只保留一个最强 Primary 行动。
- 危险按钮默认不与品牌主按钮并排争夺注意力。
- 卡片内操作优先使用 Secondary 或文字按钮，避免过度视觉干扰。

## 8. 卡片规范

### 帖子卡片

结构建议：
- 顶部：匿名头像、匿名昵称、发布时间
- 中部：帖子标题、正文摘要
- 底部：点赞数、评论数、查看详情或进入交互

设计规范：
- 背景：`#FFFFFF`
- 圆角：`12px`
- 内边距：`16px`
- 阴影：`Card Shadow`
- 标题：`H3`
- 摘要：`Body`
- 元信息：`Caption`

交互要求：
- 整卡可点击
- 桌面端 Hover 提升阴影并轻微上浮
- 摘要最多显示 3 行，标题最多显示 2 行

### 评论卡片

结构建议：
- 顶部：匿名头像、昵称、时间
- 中部：评论正文
- 底部：点赞操作

设计规范：
- 背景：`#FFFFFF`
- 圆角：`8px`
- 内边距：`12px`
- 边框优先于重阴影
- 正文字号：`16px`
- 辅助信息：`13px`

交互要求：
- 比帖子卡片更轻量
- 评论区列表通过间距区分，不依赖强分割线

### 用户信息卡片

结构建议：
- 头像或匿名标识
- 用户 ID
- 注册时间
- 发帖数、评论数等统计信息

设计规范：
- 背景：`#FFFFFF`
- 圆角：`12px`
- 内边距：`24px`
- 阴影：`Card Shadow`
- 统计信息按 2 到 3 个分区排布

交互要求：
- 以信息展示为主，弱化社交展示感
- 设置、我的帖子、我的评论等入口可作为独立行项目承接

---

## 9. 图标规范

### 推荐图标库

- 推荐使用：`Element Plus Icons`

### 图标尺寸规范

- `16px`：按钮内小图标、表单提示、状态说明
- `20px`：常规列表、卡片元信息、导航辅助图标
- `24px`：顶部导航、底部 Tab、卡片主操作图标
- `32px`：空状态、结果反馈、大模块插图型图标

### 使用规则

- 同一模块图标风格必须统一，优先线性风格。
- 图标默认跟随文字色或次文本色，不单独抢视觉焦点。
- 点赞、警告、成功等状态图标允许使用语义色。
- 图标与文字间距默认使用 `8px`。

---

## 10. 动效规范

### Hover
- 动效内容：阴影增强、轻微上浮、背景色变化
- 动画时长：`200ms`
- 使用场景：按钮、帖子卡片、桌面端导航项

### Click
- 动效内容：轻微缩放或按压反馈
- 动画时长：`120ms`
- 使用场景：按钮点击、点赞、评论发送、Tab 切换

### Page Transition
- 动效内容：淡入 + 轻微位移
- 动画时长：`240ms`
- 使用场景：首页切详情、登录切注册、用户中心子页切换

### Loading
- 动效内容：旋转、骨架屏、渐变脉冲
- 动画时长：`800ms - 1200ms` 循环
- 使用场景：帖子流加载、评论加载、提交中状态

### 动效使用规则

- 所有交互动效都应短、轻、明确，不做炫技型大幅运动。
- 移动端优先减少位移动画，保留透明度和状态变化。
- 危险操作弹窗的出现可略慢于普通提示，以增强确认感。

## 11. CSS 变量规范

设计系统必须沉淀到统一的 `:root` 变量体系中，建议按以下结构组织。

### 颜色变量

- `--color-primary`
- `--color-secondary`
- `--color-success`
- `--color-warning`
- `--color-danger`
- `--color-info`
- `--color-text-primary`
- `--color-text-secondary`
- `--color-text-disabled`
- `--color-bg-page`
- `--color-bg-card`
- `--color-bg-hover`
- `--color-border`

### 间距变量

- `--space-4`
- `--space-8`
- `--space-12`
- `--space-16`
- `--space-24`
- `--space-32`
- `--space-48`

### 圆角变量

- `--radius-4`
- `--radius-8`
- `--radius-12`
- `--radius-16`

### 阴影变量

- `--shadow-card`
- `--shadow-hover`
- `--shadow-modal`

### 变量治理规则

- 所有组件禁止直接写散值，必须优先消费设计令牌。
- 后续若有深色模式或品牌升级，仅允许调整令牌层，不直接改业务组件样式规则。
- 与 Element Plus 主题变量映射时，CampusTree 自定义变量应作为品牌主源。

## 12. 组件级落地建议

### 导航系统
- 顶部导航高度建议：桌面端 `60px`
- 移动端底部导航高度建议：`56px`
- 选中态使用 `Primary`
- 非选中态使用次文本色

### 输入与表单
- 输入框默认圆角：`8px`
- 输入框高度建议：`40px`
- 聚焦态使用 `Primary` 边框
- 错误态使用 `Danger` 边框和说明文字

### 状态反馈
- 成功：`Success`
- 警告：`Warning`
- 错误：`Danger`
- 中性说明：`Info`

### 空状态
- 使用 `32px` 图标
- 主文案使用 `Body`
- 辅文案使用 `Caption`
- 与主操作按钮之间保持 `16px` 以上间距

---

## 13. 响应式建议

### 移动端
- 页面边距：`16px`
- 卡片内边距：`16px`
- 一级标题可降级显示
- 底部 Tab 作为核心导航

### 平板端
- 页面边距：`24px`
- 保留主内容优先，侧栏可折叠

### 桌面端
- 页面边距：`24px - 32px`
- 支持卡片 Hover
- 可出现三栏布局，但主内容列依然保持阅读宽度控制

---

## 14. 最终结论

CampusTree 的 Design System 应呈现为一套清新、克制、移动优先、匿名友好的界面标准。它不是为了制造强品牌压迫感，而是为了让内容表达、情绪共鸣和轻互动更自然地发生。

这套系统的核心落点是：

- 用绿色建立品牌识别和积极校园感
- 用卡片、留白和中性色建立清晰层级
- 用统一的设计令牌支撑前端实现与长期扩展
- 用克制的动效和轻量交互保护匿名社区氛围
